# L3 兜底: 执行清单（当 executor Agent 降级失败时使用）

> 当 L1(sonnet) 和 L2(haiku) executor 均失败时，按以下清单手动执行。

## 执行前检查
- [ ] 已读取所有需要修改的文件
- [ ] 已理解现有代码逻辑和调用关系
- [ ] 已确认修改范围（列出具体文件和行号）
- [ ] 已评估对其他模块的影响

## 执行步骤
1. **最小变更原则**: 只改必须改的，不做额外重构
2. **单文件单步**: 每次只修改一个文件，改完立即验证
3. **验证顺序**: 语法检查 → 类型检查 → 单元测试 → 构建
4. **回滚准备**: 每步修改前确认可用 `git checkout -- <file>` 回滚

## 验证清单
- [ ] `bash -n <script>` 或 `tsc --noEmit` 通过
- [ ] 相关测试通过
- [ ] `git diff` 确认变更范围符合预期
- [ ] 无新增 TODO/FIXME/HACK

## 失败处理
- 如果修改后测试失败 → `git checkout -- <file>` 回滚 → 报告给用户
- 禁止在 L3 模式下尝试超过 3 次修改同一文件
