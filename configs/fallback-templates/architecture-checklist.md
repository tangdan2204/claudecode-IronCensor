# L3 兜底: 架构决策树（当 architect Agent 降级失败时使用）

> 当 L1(opus) 和 L2(sonnet) architect 均失败时，按以下决策树评估。

## 快速架构评估

### 变更范围判定
```
修改文件数:
  ≤2 → 局部修改，无需架构审查
  3-10 → 检查是否跨模块，是则需要接口审查
  >10 → 高风险，必须用户确认后再执行
```

### 模式选择决策树
```
是否涉及新 API 端点？
  是 → 使用项目已有的 API 模式（REST/GraphQL），不引入新范式
  否 ↓

是否涉及状态管理？
  是 → 使用项目已有的状态管理方案，不引入新库
  否 ↓

是否涉及文件结构？
  是 → 遵循项目现有目录约定，不创建新的顶层目录
  否 → 按现有代码风格实现
```

### 安全决策
```
是否涉及用户输入？ → 必须验证/转义
是否涉及文件操作？ → 必须检查路径遍历
是否涉及外部 API？ → 必须有超时和错误处理
是否涉及认证/授权？ → 不做任何修改，报告给用户
```

## 默认原则
1. **保守优先**: 不确定时选择更简单、更保守的方案
2. **遵循先例**: 优先使用项目中已有的模式和库
3. **用户决策**: 架构级决策必须由用户确认，L3 模式不做架构决策
